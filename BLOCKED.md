# 자율 모드 차단 사유

## Phase 2-9: File 업로드 (이미지)

**상태**: BLOCKED — 외부 정보 필요

### 필요한 결정
- **파일 저장 정책**: 다음 중 하나
  - (A) 가비아 g클라우드 VM 로컬 디스크 (`/var/uploads/...`) + nginx static serve
  - (B) AWS S3 또는 NCP Object Storage 등 외부 객체 스토리지
  - (C) DB BLOB 저장 (이미지 사이즈 제한)
- **이미지 크기 제한** + **허용 MIME**
- **CDN 사용 여부** (가비아 자체 CDN? Cloudflare?)
- **가비아 VM 디스크 용량** (`1.201.126.43` 의 가용 GB)

### 자율 모드가 결정하지 못하는 이유
사용자의 가비아 호스팅 정책 + 비용 제약 + DNS/CDN 운영 방침이 외부 정보. 메모리에 가비아 VM IP는 있지만 디스크 공간/CDN 정책 정보 없음. 임의 선택 시 운영 전환 시 재작업 발생 위험.

### 부분 진행 가능 옵션
사용자 결정 전이라도 다음은 진행 가능:
- `MultipartFile` 받는 컨트롤러 시그니처 + 검증 (size/MIME) — 저장 위치만 placeholder
- `application-local.yml`에 `app.upload.dir=__pending_user_value__` 추가
- 그러나 reviewer가 "저장 위치 미정 → 통합 테스트 불가" 이유로 FAIL 위험

### 권장 처리
사용자가 일어나면 위 결정 중 하나 답변. 답변 받은 후 1-2시간 단위로 구현 가능.

→ **이 단위는 2-10 (등급/포인트) 다음으로 미루고 진행**.
