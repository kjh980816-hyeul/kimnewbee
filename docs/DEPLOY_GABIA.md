## 0. 준비물

- [ ] **신분증 or 휴대폰**: 본인 인증
- [ ] **결제수단**: 신용카드 또는 계좌

## 1. 가비아 회원가입 + 결제수단 등록

> 이미 가입한 상태면 → 1-3만 확인하고 §2로.

### 1-1. 회원가입

1. https://www.gabia.com 접속
2. 우상단 **회원가입** 클릭
3. **개인 회원** 선택 → 약관 동의
4. 본인 인증 (휴대폰 인증)
5. 아이디/비번 정하고 가입 완료

### 1-2. 본인 인증 확인

- My가비아 → 회원정보 → 본인 인증 상태 **인증 완료** 인지 확인
- 클라우드 서비스는 본인 인증 완료된 계정만 사용 가능

### 1-3. 결제수단 등록

- My가비아 → **결제수단 관리** (또는 **My가비아 > 결제관리**)
- 신용카드 또는 계좌이체 등록
- 카드 등록 시 **자동결제 동의** 체크해두면 매월 끊김 없이 결제됨

## 2. g클라우드 서비스 신청 (약관 동의)

가비아 회원이어도 g클라우드는 **별도 약관 동의** 필요.

1. https://www.gabia.com 로그인
2. 상단 메뉴 **클라우드** → **g클라우드**
3. **서비스 신청** 버튼 → 약관 동의 → 신청 완료
4. 같은 페이지에 **콘솔 바로가기** 버튼 생김 → 클릭해서 g클라우드 콘솔 진입

> 💡 콘솔 진입이 안 보이면 **My가비아 > 클라우드** 메뉴에서도 들어갈 수 있음.

## 3. g클라우드 서버(VM) 만들기 → 공인 IP 받기

여기서 만드는 게 **인터넷에 항상 떠있는 컴퓨터 한 대**.

1. g클라우드 콘솔 좌측 메뉴 **서버 > 서버 관리** → 우측 상단 **서버 생성** 버튼
2. 다음 옵션을 선택:

| 항목 | 선택 값 | 비고 |
|---|---|---|
| **존(Zone)** | KR-Central-A 또는 비슷한 한국 존 | DB도 같은 존에 만들 것 |
| **이미지(OS)** | **Ubuntu 22.04 LTS** | 자료 가장 많고 안정적 |
| **상품(사양)** | vCPU 1 / RAM 2GB / SSD 50GB 정도 (예: g1.c1m2) | 작게 시작 → 부족하면 업 |
| **요금제** | **시간 종량제** | 처음엔 종량제 추천 (끄면 과금 안 됨) |
| **서버 이름** | `gochubat-web` | 식별용 |
| **공인 IP** | **사용** 체크 ★ | 인터넷 접근하려면 필수 |
| **방화벽** | 일단 기본값 (§5에서 따로 설정) | |
| **SSH 키 / 비밀번호** | **비밀번호 인증** + 강한 비번 (15자 이상) | 메모 필수 |

3. **신청** 클릭 → 1~3분 대기 → 상태가 **Running** 으로 바뀌면 완성

### 3-1. 받은 정보 메모 ★

서버 생성 후 콘솔 상세 화면에서 다음을 메모장에 적어둔다:

```
[ VM 정보 ]
공인 IP: 211.xxx.xxx.xxx       ← §6, §10에서 사용
내부 IP: 10.xxx.xxx.xxx        ← (있으면) DB 연결용
SSH 사용자명: root or ubuntu   ← 콘솔 "접속 정보" 화면에 표시
SSH 포트: 22 (기본)
비밀번호: ___________________  ← §3에서 본인이 정한 거
존: KR-Central-A (예시)
```

> ⚠️ 비밀번호 잃어버리면 콘솔에서 **비밀번호 재설정** 해서 다시 받을 수 있음. 그래도 처음부터 메모하자.

## 4. (선택) g클라우드 DB 인스턴스 만들기

DB를 어디에 둘지 두 가지 옵션:

### A안: 위 VM 안에 MySQL 직접 설치 (Claude가 처리)

- **추가 비용 X.** §5로 바로 넘어가면 됨.
- Claude가 SSH 접속해서 MySQL 설치/설정함.

### B안: g클라우드 DB 인스턴스 별도 생성

- **월 1만원대 추가**. 백업/모니터링 자동.
- 다음 절차:

1. g클라우드 콘솔 좌측 **DB > MySQL** (또는 **DB 서비스**) 메뉴
2. **DB 인스턴스 생성** → 옵션:

| 항목 | 선택 값 |
|---|---|
| **버전** | MySQL 8.0 |
| **존** | VM과 같은 존 (예: KR-Central-A) |
| **사양** | 가장 작은 것 (1 vCPU / 1~2GB) |
| **DB 이름** | `gochubat` |
| **마스터 사용자** | `gochubat_admin` (메모) |
| **마스터 비밀번호** | 강한 비번 16자 이상 (메모 필수) |
| **공인 IP** | **사용 안 함** (보안) |

3. 생성 후 **내부 IP / 호스트 주소** 메모 (예: `10.x.x.x`)
4. DB 인스턴스 → **방화벽 / 접근 제어** 메뉴 → **VM의 내부 IP** 만 허용

### B안 메모 ★

```
[ DB 정보 (B안 선택 시) ]
호스트(내부): 10.xxx.xxx.xxx
포트: 3306
DB 이름: gochubat
마스터 사용자: gochubat_admin
마스터 비밀번호: ___________________
```

> **A안/B안 어느 쪽 선택했는지** §10에서 Claude에게 알려줘야 함.

## 5. 방화벽(보안 그룹) 포트 열기

서버는 만들었지만 인터넷에서 들어올 수 있는 포트가 닫혀있다. 우리가 쓸 포트만 연다.

g클라우드 콘솔 → **서버 > 방화벽** 또는 **네트워크 > 보안그룹** 메뉴 → 위에서 만든 서버에 적용된 그룹 편집:

| 포트 | 프로토콜 | 출발지 | 용도 |
|---|---|---|---|
| **22** | TCP | **본인 집 IP만** (예: `121.131.xx.xx/32`) | SSH 접속 |
| **80** | TCP | `0.0.0.0/0` (전체 허용) | HTTP |
| **443** | TCP | `0.0.0.0/0` (전체 허용) | HTTPS |

본인 집 IP 확인: 네이버에서 "내 IP" 검색 또는 https://www.whatismyip.com/

> ⚠️ **22번 포트를 0.0.0.0/0으로 열어두지 마라.** 전 세계의 봇이 24시간 무차별 비번 공격함. 본인 IP만 허용.

> ⚠️ Claude가 SSH 접속하려면 **Claude가 사용하는 PC의 IP도 허용**해야 한다 — 네 PC에서 작업하니까 어차피 같은 IP로 봐도 됨. 다른 PC에서 한다면 그 IP도 추가.

> ⚠️ **3306(MySQL), 8080(Spring) 은 절대 인터넷에 열지 말 것.** 서버 내부에서만 통신.

### 5-1. 메모 ★

```
[ 방화벽 ]
22 → 본인 IP (121.xxx.xxx.xxx/32)
80 → 0.0.0.0/0
443 → 0.0.0.0/0
```

## 6. 도메인 DNS를 새 서버 IP로 연결

이미 산 도메인을 §3에서 받은 **공인 IP** 로 연결.

1. **My가비아 > 서비스 관리 > 도메인 통합관리툴** 진입
2. 본인 도메인 옆 **DNS 정보 변경** 또는 **관리 > DNS 레코드** 클릭

   > 만약 도메인이 가비아 기본 네임서버가 아닌 다른 곳을 쓰고 있으면 가비아 콘솔에선 DNS 편집이 안 보일 수 있다. 그땐 **네임서버를 가비아 기본**(`ns.gabia.co.kr`, `ns1.gabia.co.kr`)으로 먼저 변경.

3. 다음 두 줄 추가/수정:

| 타입 | 호스트 | 값/위치 | TTL |
|---|---|---|---|
| **A** | `@` | (§3에서 받은 공인 IP, 예: `211.234.123.45`) | 3600 |
| **A** | `www` | (같은 IP) | 3600 |

기존에 다른 A 레코드가 있으면 (다른 사이트 가리키고 있으면) 새로 가는 거 맞는지 확인 후 덮어쓰기.

4. **저장** 클릭
5. 5분~24시간 대기 (보통 5~30분 안)

확인 방법 (본인 PC PowerShell):
```powershell
nslookup gochubat.kr
```
→ 결과의 `Address` 가 §3 공인 IP면 OK.

### 6-1. 메모 ★

```
[ 도메인 ]
도메인: gochubat.kr (예시)
A 레코드 → §3 공인 IP
DNS 적용 확인: O / X
```

## 7. 네이버 개발자센터에 운영 도메인 등록

네이버 OAuth 콜백이 운영 도메인으로도 작동하도록 등록.

1. https://developers.naver.com 로그인
2. 상단 **Application > 내 애플리케이션** → 본인 앱 선택 → **API 설정**
3. **로그인 오픈 API 서비스 환경** 섹션에서 **PC 웹** 항목 추가:
   - **서비스 URL**: `https://gochubat.kr` (본인 도메인)
   - **Callback URL**: `https://gochubat.kr/api/auth/naver/callback`
4. **수정** 버튼 클릭

> 기존에 등록된 `localhost:5173` / `localhost:8080` 항목은 **그대로 둬도 됨** (개발용).

### 7-1. 메모 ★

```
[ 네이버 OAuth ]
Client ID: __________________
Client Secret: __________________  ← Claude에게 줄 것
운영 서비스 URL: https://gochubat.kr
운영 Callback URL: https://gochubat.kr/api/auth/naver/callback
```

> ⚠️ **Client Secret 은 시크릿이다.** 이전에 채팅에 노출된 적 있으면 **재발급** 후 새 값 사용.

## 8. SSH 접속 정보 / 비밀번호 정리

§3 비밀번호와 §3 공인 IP가 잘 메모됐는지 마지막 확인.

직접 SSH 접속 한 번 시도해보면 깔끔:

PowerShell에서:
```powershell
ssh root@(공인_IP)
```
또는
```powershell
ssh ubuntu@(공인_IP)
```

비번 입력해서 들어가지면 OK. 들어가지면 바로 `exit` 로 빠져나오기 — 안에서 뭐 만지지 말 것 (그건 Claude가 함).

> 안 들어가지면:
> - `Connection refused` → §5 방화벽 22 포트 본인 IP 다시 확인
> - `Permission denied` → 비번 또는 사용자명 틀림. 가비아 콘솔 → 서버 상세 → **접속 정보** 화면에 표시된 사용자명 다시 확인
